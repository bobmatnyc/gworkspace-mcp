# get_document returns empty content for modern tabbed Google Docs

## Description
The get_document MCP tool intermittently returned empty content for Google Docs documents. The document title was present but text_content was empty.

Root cause: Google Docs API moved content from body.content to tabs[].documentTab.body for modern tabbed documents. The tool was reading from the old body path (empty for tabbed docs) and only conditionally requesting includeTabsContent=true.

Fix (committed as d0cf9d4):
1. Always send includeTabsContent=true to the Docs API
2. Extract text from tabs[].documentTab.body when tabs are present
3. Fall back to body for legacy single-tab documents

Affected file: src/gworkspace_mcp/server/services/docs/core.py
